# Habit Fork

Загляни в своё будущее по привычкам. Вводишь свои привычки → AI показывает, кем ты
станешь через 10 лет, и **развилку двух судеб**: если продолжишь vs если изменишь 3
привычки.

> Хакатон-прототип. Число «корреляция к успеху» — это **проекция AI / зеркало**, а не
> научное предсказание.

## Что есть сейчас

- `index.html` — рабочий фронт без зависимостей (открывается двойным кликом).
  - Экран ввода: привычки (3–20) + опциональное фото.
  - Геройский экран: число + драйверы (▲/▼) + Habit Clash + диптих двух будущих.
  - Живой мок: типизация привычек по ключевым словам реально двигает результат.

## Как запустить

**Без бэкенда (быстрый просмотр):** открой `index.html` в браузере. Всё считается локально
в JS-моке.

**С бэкендом (настоящий AI):**
```bash
pip install -r requirements.txt
cp .env.example .env          # впиши ANTHROPIC_API_KEY и HF_TOKEN
uvicorn app:app --reload --port 8000
# открой http://localhost:8000
```
Без ключей бэкенд тоже работает — на эвристике (число считается, картинки = плейсхолдер).
Проверить состояние: `GET /api/health` → `{llm, images}`.

**Telegram-бот (второй канал, тот же бэкенд):**
```bash
pip install -r requirements.txt
# в .env добавь TELEGRAM_BOT_TOKEN (возьми у @BotFather)
python bot.py
```
Бот принимает привычки сообщением → шлёт одну склеенную карточку (диптих + число + clash +
3 привычки) с кнопками «Заново» и «Поделиться». Карточка самодостаточна — её форвардят в чат.

## Архитектура

```
habits[] ─► llm.parse_habits ─► scoring.score_habits ─► images.generate_pair ─► JSON ─► фронт
            (Claude / эвристика)  (формула 0..100)        (SDXL-Turbo, параллель)
```

| Кусок | Файл | Статус |
|-------|------|--------|
| Парсинг + clash + промпты сцен | `backend/llm.py`, `backend/prompts.py` | код готов, нужен `ANTHROPIC_API_KEY` (иначе эвристика) |
| Скоринг 0–100 | `backend/scoring.py` | готово, детерминированно |
| Картинки развилки | `backend/images.py` | код готов, нужен `HF_TOKEN` (иначе плейсхолдер) |
| Веб-фронт | `index.html`, `app.py` | готово (FastAPI + JS-фолбэк) |
| Telegram-бот | `bot.py`, `backend/card.py` | код готов, нужен `TELEGRAM_BOT_TOKEN` |
| Шер-карточка (PNG) | `backend/card.py` | готово для бота; для веба — TODO |

Два канала, один бэкенд: веб (`app.py`) — красивый экран на ноуте; бот (`bot.py`) —
виралка и онлайн-AI. `llm` / `scoring` / `images` общие.

Главный риск демо — латентность генерации картинок. Сейчас: 2 вызова параллельно + фолбэк на
плейсхолдер. Дальше: пред-генерёжка демо-профиля + прогрев эндпоинта.

## Стек

Фронт: ванильный HTML/CSS/JS. LLM: hosted API. Картинки: SDXL-Turbo (Hugging Face).
Один горизонт в демо: +10 лет.
